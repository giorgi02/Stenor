# Stenor

სწრაფი, მინიმალისტური კარნახის (dictation) პროგრამა Windows-ისთვის — Wispr Flow-ის
ალტერნატივა. დაიჭირე ღილაკი, ილაპარაკე, აუშვი: ნათქვამს Google Gemini ტექსტად გარდაქმნის და
ჩაწერს იმ აპლიკაციაში, რომელიც ამ მომენტში ფოკუსშია (VS Code, Word, ბრაუზერი, Slack, Teams…).

- **სტეკი:** C# / .NET 10, WPF (მხოლოდ პარამეტრები + ოვერლეი), NAudio (WASAPI),
  Google.GenAI SDK, H.NotifyIcon (ტრეი), Velopack (განახლებების კარკასი), ხელით დაწერილი
  Win32 P/Invoke.
- **მოდელი:** `gemini-3.1-flash-lite`
- **გაზომილი ამ კომპიუტერზე (published build):** ცივი სტარტი ≈ 1.2 წმ, უქმე რეჟიმში
  working set ≈ 6 MB (57 MB private), უქმე CPU 0%.

## აწყობა და გამოქვეყნება

```powershell
# ჩვეულებრივი აწყობა
dotnet build Stenor.slnx -c Release

# საპროდაქშენო publish (self-contained, ReadyToRun)
dotnet publish src/Stenor.App/Stenor.App.csproj -c Release -r win-x64 --self-contained
# → src/Stenor.App/bin/Release/net10.0-windows10.0.19041.0/win-x64/publish/Stenor.exe
```

აწყობა გაფრთხილებების (warnings) გარეშეა და ამას `TreatWarningsAsErrors` აკონტროლებს.

## პირველი გაშვება

Stenor სისტემურ ტრეიში ცხოვრობს (taskbar-ზე ფანჯარა არ აქვს). პირველ გაშვებაზე პარამეტრების
ფანჯარა თავად იხსნება — ჩასვი Gemini API გასაღები <https://aistudio.google.com/apikey>-დან,
სურვილისამებრ დააჭირე **Test key**-ს და შეინახე. შემდეგ გადადი ნებისმიერ ტექსტურ ველში,
**დაიჭირე მარჯვენა Ctrl**, ილაპარაკე და აუშვი.

პარამეტრები (ტრეის ხატულაზე ორმაგი კლიკი ან მენიუში *Settings…*):

| პარამეტრი | ნაგულისხმევი | შენიშვნა |
|---|---|---|
| Gemini API გასაღები | — | ინახება DPAPI-თ დაშიფრული (CurrentUser) `%APPDATA%\Stenor\settings.json`-ში |
| ძირითადი ენა | ინგლისური | მხოლოდ *მინიშნებაა* ტრანსკრიფციისთვის, სხვა ენებიც მუშაობს |
| ღილაკი (hotkey) | მარჯვენა Ctrl | ცალკეული ღილაკები (მარცხენა/მარჯვენა განირჩევა) ან კომბინაციები |
| აქტივაციის რეჟიმი | Hold | Hold = push-to-talk, Toggle = დაჭერა/დაჭერა |
| Windows-თან ერთად გაშვება | ჩართული | HKCU `…\CurrentVersion\Run` |

ლოგები: `%APPDATA%\Stenor\logs\` (როტაციით; არასდროს შეიცავს აუდიოს, ტრანსკრიპტებს ან გასაღებს).

## როგორ მუშაობს

**მდგომარეობათა მანქანა** (`DictationController`): Idle → Recording → Transcribing →
Injecting → Idle. სანამ სისტემა დაკავებულია, ღილაკზე დაჭერები იგნორირდება, ამიტომ სწრაფი
ორმაგი აქტივაცია უსაფრთხოა. ყველა ჩავარდნის გზა Idle-ში ბრუნდება ოვერლეის/ტრეის
შეტყობინებით — სერვისები იზოლირებულია და აპლიკაცია არასდროს იშლება (გლობალური exception
ჰენდლერები ლოგავენ და აგრძელებენ მუშაობას).

**გლობალური ღილაკი** (`HotkeyService`): `WH_KEYBOARD_LL` დაბალი დონის hook ცალკე
message-pump ნაკადზე. Callback-ი თითქმის ნულოვან სამუშაოს ასრულებს — გამოტოვებს ინექცირებულ
მოვლენებს, აახლებს მოდიფიკატორების ბიტმასკას, `TryWrite`-ით წერს შემოსაზღვრულ `Channel`-ში
და ბრუნდება — რადგან ნელი LL hook მთელი სისტემის კლავიატურას აყოვნებს და Windows მას ჩუმად
შლის. შესატყვისობის/დებაუნსის/რეჟიმის მთელი ლოგიკა ცალკე consumer ტასკზე მუშაობს. შიშველი
მოდიფიკატორი (მაგ. მარჯვენა Ctrl) სისტემას არასდროს ეპარება — გადაეცემა OS-ს; კომბინაციებში
მხოლოდ მთავარი ღილაკი ითრგუნება. 150 მწმ-ზე მოკლე დაჭერები შემთხვევით შეხებად ითვლება და
უგულებელყოფილია.

**აუდიო** (`RecorderService`): ერთი `WasapiCapture` ინსტანცია იქმნება და „თბება" (ერთი
სწრაფი start/stop ციკლი) გაშვებისას, ამიტომ რეალური ჩაწერა 50 მწმ-ზე ბევრად სწრაფად იწყება,
ხოლო მიკროფონის ინდიკატორი უქმე რეჟიმში გამორთული რჩება. აუდიო იწერება მოწყობილობის მშობლიურ
ფორმატში და პირდაპირ გარდაიქმნება (მონოში დაყვანა → WDL რესემპლინგი) 16 კჰც/16-ბიტიან მონო
WAV-ად მეხსიერებაში. ჩაწერა 5 წუთზე ავტომატურად ჩერდება. მიკროფონის გამოერთება/შეცვლა
აღმოჩენადი და აღდგენადია (ტრეის შეტყობინება, არასდროს — კრახი).

**ტრანსკრიფცია** (`TranscriptionService`): Google.GenAI SDK, აუდიო ინლაინ `audio/wav`-ად +
მკაცრი პრომპტი (სიტყვასიტყვით, „ეე"-ების მოშორება, პუნქტუაცია, მხოლოდ ტექსტი) და ენის
მინიშნება. 30 წმ timeout, ერთი ავტომატური გამეორება 5xx/ქსელის შეცდომებზე; ავტორიზაციის
შეცდომები მაშინვე გასაგები ტექსტით ჩნდება.

**ტექსტის ჩასმა** (`InjectionService`): ბუფერის (clipboard) სარეზერვო ასლი
(ტექსტი/სურათი/ფაილები) → ტრანსკრიპტის ჩაწერა → `SendInput` Ctrl+V → ბუფერის აღდგენა ~300
მწმ-ში. ინექცირებულ ღილაკებს აქვთ სპეციალური `dwExtraInfo` ნიშანი („STNR"), რომელსაც hook-ი
იგნორირებს; ფიზიკურად დაჭერილი მოდიფიკატორები (მაგ. hotkey-ის საკუთარი Ctrl Toggle რეჟიმში)
ჯერ „აეშვება", რომ ჩასმის აკორდი არ დაზიანდეს. არსებობს სიმბოლო-სიმბოლო
`KEYEVENTF_UNICODE` fallback-იც (`UseUnicodeTypingFallback` settings.json-ში; ბუფერის
დაბლოკვისას ავტომატურადაც ირთვება).

**ოვერლეი** (`OverlayWindow`): უჩარჩოო მუქი „ბალიში" `WS_EX_TOOLWINDOW | WS_EX_NOACTIVATE`
სტილებით — ფოკუსს ვერასდროს წაართმევს, რაც ჩასმის სამიზნეს გააფუჭებდა. ჩნდება იმ მონიტორის
ქვედა ცენტრში, სადაც ფოკუსირებული ფანჯარაა (შერეული DPI-ის მხარდაჭერით —
`GetDpiForMonitor` + `SetWindowPos` პიქსელებში). მდგომარეობები: 🎙 ჩაწერა (ხმის დონის
ცოცხალი ზოლები) → ⏳ ტრანსკრიფცია → ✓ დასრულდა (1 წმ) / ⚠ შეცდომა (3 წმ). ✕ აუქმებს ჩაწერას
ან ტრანსკრიფციას.

**ერთი ინსტანცია:** სახელიანი mutex `Stenor.SingleInstance`; მეორედ გაშვება სიგნალს უგზავნის
მომუშავე ინსტანციას (პარამეტრები იხსნება) და გამოდის.

**განახლებები:** Velopack ჩაშენებულია (`VelopackApp.Build().Run()` + ფონური შემოწმება),
მაგრამ უმოქმედოა, სანამ settings.json-ში `UpdateFeedUrl` არ მიეთითება — feed-ის გარეშე
ქსელურ მოთხოვნებს არ აკეთებს და არასდროს აბლოკირებს.

**მეხსიერება:** გაშვების დასრულებისა და პარამეტრების ფანჯრის დახურვის შემდეგ (ფანჯარა
ნადგურდება და არა იმალება) GC ტკეპნის მეხსიერებას და working set-ი იჭრება — უქმე კვალი
პატარა რჩება.

      ოვერლეი იმავე მონიტორზე ჩნდება ფოკუსის წართმევის გარეშე; ტექსტი აპლიკაციაში ჯდება.
- [ ] **სურათი ბუფერში** კარნახის წინ: უკარნახე; ~300 მწმ-ში სურათი ბუფერში ბრუნდება
      (Paint-ში ჩასვი შესამოწმებლად).
- [ ] **სწრაფი ორმაგი აქტივაცია:** ღილაკს სწრაფად აჭირე — ჩაწერა ან მუშაობს, ან უქმდება
      (150 მწმ-ზე მოკლე შეხებები); არავითარი კრახი, გაჭედილი მდგომარეობა ან გაორებული ჩაწერა.

      აგრძელებს მუშაობას; მიკროფონის დაბრუნების შემდეგ კარნახი ისევ მუშაობს.
- [ ] **ნაგულისხმევი მიკროფონის შეცვლა** უქმე რეჟიმში: შემდეგი კარნახი ახალ მოწყობილობას იყენებს.
- [ ] **არასწორი API გასაღები:** გასაგები შეცდომა ოვერლეიზე + შეტყობინება, გამეორებების

- [ ] **Toggle რეჟიმი:** დაჭერა იწყებს, მეორე დაჭერა აჩერებს; ღილაკის დაჭერილად დატოვება
      განმეორებით არ რთავს/თიშავს (auto-repeat გაფილტრულია).

### სხვა ცნობილი შეზღუდვები

- **Win** ღილაკიანი კომბინაციების ჩაწერისას OS-ის მალსახმობები მაინც შეიძლება ჩაირთოს.
- ინექცირებული/სინთეზური კლავიშები (AutoHotkey და მისთ.) hotkey-ს განზრახ არ ააქტიურებენ.
- თუ ბუფერში ტექსტის/სურათის/ფაილების სიის გარდა აპლიკაციის კერძო ფორმატები იყო, ჩასმის
  შემდეგ მხოლოდ ეს სამი აღდგება.
