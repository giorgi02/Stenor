# ხელით შესამოწმებელი სია

ავტომატური ტესტები არ გვაქვს — ცვლილებების შემდეგ ეს სცენარები ხელით მოწმდება
(published build-ზე).

- [ ] **სრულეკრანიანი აპლიკაცია ფოკუსში** (თამაში/ვიდეო): დაიჭირე ღილაკი, ილაპარაკე —
      ოვერლეი იმავე მონიტორზე ჩნდება ფოკუსის წართმევის გარეშე; ტექსტი აპლიკაციაში ჯდება.
- [ ] **სურათი ბუფერში** კარნახის წინ: უკარნახე; ~300 მწმ-ში სურათი ბუფერში ბრუნდება
      (Paint-ში ჩასვი შესამოწმებლად).
- [ ] **სწრაფი ორმაგი აქტივაცია:** ღილაკს სწრაფად აჭირე — ჩაწერა ან მუშაობს, ან უქმდება
      (150 მწმ-ზე მოკლე შეხებები); არავითარი კრახი, გაჭედილი მდგომარეობა ან გაორებული ჩაწერა.
- [ ] **მიკროფონის გამოერთება ჩაწერისას:** შეტყობინება + ოვერლეის შეცდომა, აპლიკაცია
      აგრძელებს მუშაობას; მიკროფონის დაბრუნების შემდეგ კარნახი ისევ მუშაობს.
- [ ] **ნაგულისხმევი მიკროფონის შეცვლა** უქმე რეჟიმში: შემდეგი კარნახი ახალ მოწყობილობას იყენებს.
- [ ] **არასწორი API გასაღები:** გასაგები შეცდომა ოვერლეიზე + შეტყობინება, გამეორებების
      შტორმის გარეშე; **Test key** პარამეტრებში უარყოფას აჩვენებს.
- [ ] **ძალიან გრძელი კარნახი:** 5 წუთზე ავტომატურად ჩერდება და ნორმალურად ტრანსკრიბირდება.
- [ ] **Toggle რეჟიმი:** დაჭერა იწყებს, მეორე დაჭერა აჩერებს; ღილაკის დაჭერილად დატოვება
      განმეორებით არ რთავს/თიშავს (auto-repeat გაფილტრულია).
- [ ] **ღილაკის ჩაწერა პარამეტრებში:** Settings → დააჭირე hotkey ველს → აკრიფე მაგ.
      Ctrl+Shift+D ან შიშველი მარჯვენა Shift; Esc აუქმებს. ჩაწერისას გლობალური ღილაკი
      შეჩერებულია.
- [ ] **მეხსიერება კარნახის შემდეგ:** Task Manager → Details → Stenor.exe; ერთი კარნახის
      შემდეგ working set ციკლისას იზრდება (~50 MB), მაგრამ ~30 წამში უქმე დონეს
      (<15 MB) უბრუნდება.
- [ ] **Stenor.exe-ის მეორედ გაშვება:** მეორე ტრეის ხატულა არ ჩნდება; მომუშავე ინსტანციის
      პარამეტრები იხსნება.
- [ ] **ადმინისტრატორის (elevated) ფანჯარა ფოკუსში:** *ცნობილი შეზღუდვა* — არაპრივილეგირებული
      პროცესი elevated ფანჯარაში ტექსტს ვერ აგზავნის (UIPI). ტრანსკრიპტი ბუფერში რჩება
      ხელით Ctrl+V-სთვის; თუ იქაც პირდაპირი ჩასმა გინდა, Stenor ადმინისტრატორად გაუშვი.
